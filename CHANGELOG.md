### Changed
- **Web 本番生成経路（Studio の PNG / MP4 / 透過 DL）を WebGL2 から WebGPU(WGSL) に配線替え** (#245 PR-A / #207 Phase 3)。`web/src/lib/orberWorker.ts` の `createGlRenderer`（orberGl.ts / GLSL）依存を撤去し、wasm の WebGPU 経路（新設の `gpu_init_offscreen(OffscreenCanvas)` → `gpu_set_render_data` → `gpu_render(t)`。#230/#231 のブラウザ WGSL 経路を Worker から使う）に置換した。**ユーザー可視の変化: Web の生成出力が #242（旧 WebGL の明るさを 1:1 移植した合成）+ #241（薄い影 s=0.2）の確定ルックに変わる**（CLI と同一の core WGSL シェーダで描くようになるため。これが Phase 3 の目的そのもの）。glyph は同梱フォント内 = core フォント経路（`glyph_char`）/ 収録外 = `generateJsGlyphSdf` → `WasmParams.glyph_sdf`（#231 と同設計）。image は worker で File → マスク RGBA（長辺 1024 に縮小）にデコードし `image_mask_*` で渡す（SDF 化は core の `image_rgba_to_sdf`。コントラスト不足は既存 sentinel `image-shape-no-contrast` に変換して Studio の文言マップを維持）。透過 export は新設の `WasmParams.transparent_background`（旧 worker の「pack header bg.a=0 上書き」の wasm 入口版。serde default false で既存呼び出しはバイト列不変）+ 新設の `gpu_render_rgba(t)`（canvas 非経由の straight-alpha readback。WebGPU canvas の alphaMode は opaque / premultiplied しかなく、旧 WebGL の「straight alpha のまま convertToBlob」が成立しないため、native readback と同形の texture → padded buffer → async map を wasm に追加）で動作維持。MP4 は従来どおり canvas → `VideoFrame`（WebCodecs、#112 の仕組み不変）。**WebGPU 非対応ブラウザは生成不可**（#207 裁定: fallback 無し）: worker が sentinel `webgpu-unsupported` で reject し、Studio が新文言 `webgpuUnsupported`（ja「お使いのブラウザは WebGPU 非対応です」/ en "Your browser does not support WebGPU"）を表示する。Worker RPC（メッセージ型・kind・payload）は不変で Studio 側の変更はエラーマップ 1 分岐のみ。main thread の present 経路（gpu-lab / AbPanel、surface format / alphaMode 選択ロジック）も不変。orberGl.ts / AbPanel / ab ハーネス一式の削除は PR-B（#245 後半）で行う
- orb 機構（orb / glyph / image、`crates/core/src/orb.wgsl`）の最外周フェードセグメントに**「薄い影」（rgb 暗化）を強度係数 `shadow_strength` 付きで再導入** (#241)。#242 完了後の実機 blink での kako-jun 裁定「旧の見た目をアルゴリズムとして採用しつつ、旧は少し単純すぎるから、複雑そうに見えるよう新のようなアレンジも薄く重ねる」への対応。機構は #242 で撤去した旧 lowp の straight-color フェード（rgb_scale = 1−u）を**そのまま係数化**した `mix(1.0, 1.0−u, s)`（新規カーブ・フィット無し）: **s=0 は #242 直後（影なし）と bit 同一**（mix(1,x,0)=1 と c×1.0=c の IEEE 恒等。CPU bitwise 全域スイープ + 実 GPU 全帯域 ±1 ピンで担保）、**s=1 は旧 lowp の rgb→0 フェードと同等の暗さ**（撤去前式の float 再現と外周帯 ±1 一致を実 GPU でピン）。影は falloff の r に乗るので全 shape で自動的に**シルエット沿い**（quad 矩形の枠は出ない）。Rim / Soft とも旧 lowp と同じく最外周セグメントのみ暗化（plateau・内側セグメントは不変 = bit 一致をテストで担保）。配線は glyph_rotate(#136) / edge_softness(#205) と同じ流儀: `AnimateOptions.shadow_strength` → `pack_render_data_for_webgl` の新引数 → pack header[13]（予約域、0..=1 クランプ）→ WGSL Params uniform（row 4）。**production 既定値は `core::animate::SHADOW_STRENGTH_DEFAULT = 0.2 に 1 箇所集約**（kako-jun が gpu-lab スライダーで実機選定・freeza session595）。チューニング足場: `WasmParams.shadow_strength`（serde default = 製品定数、0.0..=1.0 の範囲外 reject・両端 inclusive）+ gpu-lab の shadow スライダー（0..1 / step 0.05。既定値は wasm export `shadow_strength_default()` で製品定数に同期＝HTML ハードコードの drift 防止。orb 機構 shape でのみ表示）。**本番 Studio には出さず、CLI フラグも増やさない**（製品は定数）。aquarelle（参照 = aquarelle crate なので対象外）・WebGL（orberGl.ts は header[13] を読まない）・present 経路は不変。s>0 では WGSL が旧 WebGL より外周フェード帯だけ僅かに暗く（= 影が乗って）見えるのが正で、ab ハーネスの vs `ab-webgl.png` 比較には影分の差が出る（`ab_harness.rs` モジュール doc に注記。WGSL ↔ CLI readback は引き続き byte-near が正）。#242 の CPU 参照ピン（中心画素・外周フェード・境界 3 点・半透明 bg・2 orb 順序）は影項込みのオラクル式に更新して production 定数で green。テスト 6 件追加（s=0 bitwise 退化 / s=0 GPU 全帯域 / s=1 旧 lowp 一致 / serde 省略 default・範囲外 reject・inclusive 境界 / header[13] クランプ）— cargo 計 41(cli) + 194(core) + 63(wasm)。なお #241 の旧スコープ（● ≡ orb の厳密退化・局所太さ medial 正規化・細線輝度正規化）は kako-jun「これでもいい」裁定で本文仕様ごと凍結（必要になったら別 Issue で再起）。
- orb 機構（orb / glyph / image、`crates/core/src/orb.wgsl`）の合成を、Skia lowp 再現（stop alpha の u8 量子化 → 最外周セグメントの rgb→0 フェード → u8 premultiply の div255 合成 → premul→straight finalize）から**旧 WebGL（`web/src/lib/orberGl.ts`）の GLSL を 1:1 移植した raw float stop alpha + straight-alpha float Source-Over** に置換 (#242)。発端は #232 の A/B blink で「WGSL が WebGL より全体に暗い・灰色の枠が見える」という kako-jun 実機観察。三者画素比較（CLI readback / ブラウザ WGSL / ブラウザ WebGL — 下記 Added の `?ab=1&abcap=1` キャプチャ + `ab_harness.rs`）で (a) **present 経路はシロ**（ブラウザ WGSL = CLI readback、518,400 チャネルサンプル（270×480 RGBA）中 6 サンプルのみ ±1。surface format / alphaMode / DPR smear / 量子化の容疑者は全部無実）、(b) **旧 WebGL が core より一様に明るい**（R+30.7 / G+23.2 / B+24.2。lowp の rgb→0 フェードが暗部を一様に沈めていた = 「全体暗化」「灰色枠」の正体）ことを特定し、kako-jun 裁定「**旧の明るさが良い**」により旧 WebGL の合成アルゴリズムを正として採用。移植は補正係数・カーブフィット一切無しの式移植（`composite_premul` → `composite_straight`、`falloff_curve` は vec2(alpha, rgb_scale) → f32 raw alpha、div255 / finalize 撤去）。orb / glyph / image は #235 統一テンプレを共有するため全 shape が新合成を通る（統一機構そのものは不変）。結果: ブラウザ WGSL vs WebGL = **2 サンプル ±1 / 518,400 の実質 byte-exact パリティ**（Apple A18 Pro / Metal、t=0・合成ソース・seed=42 / n=12 / spec_idx=8。CLI readback vs ab-webgl.png も同 2 サンプル ±1 で、ゲート max±2 / mean_abs<0.5 を大差で通過）。**aquarelle（`orb_aquarelle.wgsl`）は参照アルゴリズムが aquarelle crate（Skia lowp）のため lowp 合成を意図的に維持**（`fn div255` / `fn composite_premul` の残存を string pin テストで保護）。`orberGl.ts` は 1 文字も不変（キャプチャ round1 vs round2 の byte-exact 一致で決定論も証明済み）。CLI byte-exact 基線（#235）は裁定どおり引き直しで、再ピンは `gpu_glyph_bullet_approx_equals_orb` の IoU 床 0.85→0.75（A18 Pro 実測 0.957→0.827。rgb→0 フェード撤去で淡い外周リングが px_lit 閾値を越える設計上の変化）の 1 件のみ。影のくっきり感・量の調整（裁定の補足「旧ベース + 新のアレンジを薄く重ねる」）は #241 の管轄。
- Studio に WebGL↔WGSL の A/B ライブ比較パネルを追加 (#232)。`?ab=1` クエリのときだけ Studio に現れる**検証専用パネル**（`web/src/components/AbPanel.tsx`）で、本番 UI と本番の生成経路（バッチ / DL）には一切触れていない。canvas を 2 枚スタックし、上に WebGL（`GlRenderer`、main thread）/ 下に WGSL（`gpu_init` / `gpu_set_render_data` / `gpu_render`）を置いて、トグルで瞬時に切替（blink 比較）する。両レンダラは同じ wall-clock `t`（周期 8000ms）で駆動するので、切替の前後で同位相のフレームを見比べられる。入力も Studio の現在状態（source 画像 / shape / glyph 文字 / count・speed・softness プリセット）を共有しつつ、再現性のために seed=42 / n=12 / spec_idx=8 を固定する（定数は `web/src/lib/abLogic.ts` に集約。共通 params は `buildAbBaseParams`、開始ガードは `abCanStart` の純関数）。GPU の init ms と各レンダラの FPS を計測表示し、`isWebGpuSupported()`（`web/src/lib/webgpu.ts`）が false の環境では WGSL 側を disabled にして WebGL のみ動かす。A/B が示す意味は shape ごとに異なる: **orb はパリティゲート**（#235 で orb 出力は byte-exact 不変なので両者一致するのが正）、**glyph / image は新旧の見比べ**（#235 で機構が SDF シルエット統一に変わったため一致しないのが正）、**aquarelle は対象外**（Studio に入らない。#233→#239 に統合 close）。このパネルは **Phase 3 で WebGL（`web/src/lib/orberGl.ts`）を撤去するときに、`AbPanel.tsx` / `lib/webgpu.ts` / `lib/abLogic.ts` / `strings.ts` の `ab*` キーごと削除する足場**で、恒久機能ではない。テスト 19 件追加（`webgpu.test.ts` 4 / `abLogic.test.ts` 10 / `strings.test.ts` 5、計 104）。本番 UI・既存の wasm / WebGL 呼び出し・生成バイト列はいずれも不変。
- ブラウザ WebGPU 経路を全 shape に配線 (#231)。#230 で orb のみだった wasm の WebGPU canvas-present 経路を glyph / image / aquarelle まで広げた。`ensure_gpu_supported_shape` を撤去し、`gpu_render` が shape 別に core の `render_packed_to_view`（orb）/ `render_frame_glyph_to_view` / `render_frame_image_to_view` / `render_frame_aquarelle_to_view` へディスパッチする（CLI と同一の分岐構造）。`WasmParams` に aquarelle の 4 パラメータ（`aquarelle_bleed` / `aquarelle_bloom` / `aquarelle_offset` / `aquarelle_halo`、いずれも既定 0.5）を追加し、aquarelle が wasm 入口に初登場した。image は新規のマスク入力（`image_mask_rgba` / `image_mask_width` / `image_mask_height`）を受け、core の `image_rgba_to_sdf`（256）で SDF 化する。`parse_shape` は aquarelle を受理するようになり、glyph も従来どおり扱う。dev ページ `web/src/pages/gpu-lab.astro` には shape 切替・glyph 文字入力（rotate トグル）・image アップロード・aquarelle スライダー 4 本を追加。WebGL 経路（`webgl_shape_id`）は不触で、aquarelle を明示的に reject し続け、`get_render_data` のバイト列も不変（既存 wasm / WebGL 呼び出しはバイト互換）。さらに glyph の JS フォールバックを WGSL 経路にも実装した: 同梱フォント（Noto Sans Symbols 2 サブセット＝記号のみ）に無いひらがな・漢字・絵文字は、Web が `generateJsGlyphSdf`（OffscreenCanvas + OS フォントスタック）で作った SDF を `WasmParams` の新フィールド `glyph_sdf` / `glyph_sdf_size` に乗せ、`resolve_orb_shape` が検証（size は `get_glyph_sdf` と同じ 16..=1024、`len == size*size`）して `OrbShape::Image` に解決する（#235 統一機構で glyph も image も同じ SDF シルエット経路）。WebGL 版 #159 と同じ「ユーザーが入れた字を尊重して描画する」フォールバックで、WGSL 経路でも記号以外の任意 Unicode を描けるようになった。`glyph_sdf` が空のときは従来どおり core フォント経路（`OrbShape::Glyph`）。
- glyph / image を orb の機構に統一し、`circle` → `orb` に改名 (#235)。**orb の挙動が真**（解析的距離 + falloff のぼやけ方が最も美しい）という方針のもと、glyph / image を「orb に別のシルエットを食わせる」形へ一般化した。WGSL は `orb_circle.wgsl` を `orb.wgsl` に改名し、距離源（DISTANCE SOURCE）だけを差し替え可能なテンプレートにして Rust 側（`orb_wgsl()` / `orb_sdf_wgsl()`）で 2 variant を文字列合成する: **orb** = 解析的な円距離（旧 `orb_circle.wgsl` と byte-exact、現実機 A18 Pro で確認）、**glyph / image** = SDF サンプル距離（回転を SDF サンプル前に適用、CONTENT_SPAN クリップ + sdf_size texel 補正を維持）。下流の falloff_curve / 3 軸呼吸 / Skia lowp premultiply 合成は全 shape 共通になり、glyph / image は orb と同じぼやけ方で描かれる（`●` は orb と同じ見た目、`▲` は三角のままぼやける）。旧 glyph 機構を撲滅: `orb_glyph.wgsl`（SDF fill + edge smoothstep）と `orb_glyph_bleed.wgsl`（blur×3 + halo + compose の 4 パス群、aquarelle 0.2 由来）を削除し、gpu.rs の bleed パイプライン / 中間テクスチャ / 関連キャッシュも撤去して glyph / image を単パス化した。「にじみ」は aquarelle shape だけの領分とし、glyph / image に独自のにじみ実装は残さない。命名は CLI `--shape circle` → `--shape orb`（alias 無し・後方互換なし）、`OrbShape::Circle` → `OrbShape::Orb`、wasm `parse_shape` / `ensure_gpu_supported_shape` の `"circle"` → `"orb"`、Web Studio の shape 選択（内部値 `'circle'` → `'orb'`、表示は従来どおり Orb / オーブ）、docs にまたがって統一した。WebGL 版（`web/src/lib/orberGl.ts`）は不触（Phase 3 で WebGL ごと削除予定）で、Studio が `'orb'` を渡しても WebGL の shape_id=0 経路に正しく落ちる。aquarelle shape は独自機構のまま対象外。
- 透過動画パスから ffmpeg.wasm を完全撤去し、**JS-only MOV muxer** に置換 (#192)。#184 で着地した PNG-in-MOV 経路は ffmpeg.wasm を実エンコードなしの muxer 役としてしか使っていなかったため、QuickTime atom tree (`ftyp`/`moov`(`mvhd`/`trak`(`tkhd`/`mdia`(`mdhd`/`hdlr`/`minf`(`vmhd`/`dinf`/`stbl`))))/`mdat`) を直接組み立てる ~280 行の `web/src/lib/movMuxer.ts` を新設し、`web/src/lib/encodeAlphaVideoWasm.ts` を薄い async ラッパーに書き換えた。撤去物: (a) `@ffmpeg/ffmpeg` / `@ffmpeg/util` / `@ffmpeg/core` dependency、(b) jsdelivr CDN 経由の ~30MB wasm 初回ロード (CF Pages 25 MiB 上限回避のための cross-origin 配信)、(c) Service Worker の `ffmpegCoreCacheFirst` 経路 + `ffmpeg-core-v<ver>` キャッシュ + `__FFMPEG_CORE_VERSION__` build stencil、(d) `Studio.tsx` のアイドル時投機プリフェッチ + `saveData` / `2g`/`3g` saver guard + `NetworkInformation` / `requestIdleCallback` 型定義、(e) `FFmpeg` シングルトン (`ffmpegSingleton` / `ffmpegLoadPromise` / `prefetchPromise` / `encodeMutex` / `toBlobURL` blob: URL 経路)、(f) `alphaEncoderState` シグナル + `alphaEncoderLoadFailed` i18n キー + 「encoder 読み込み失敗で alpha 一式を諦める」UI 分岐。`alphaEncodingInProgress` 文言は将来の再利用枠として温存。出力形式は #184 と同じ `alpha/*-alpha.mov` (PNG codec / rgba 32bit / 540×960 / 24fps × 8sec)、消費側コードと既存 ZIP 配置はバイト互換。Service Worker の `activate` ハンドラは現行 `orber-{date}` 以外を全掃除する形に簡略化されているため、旧 `ffmpeg-core-v*` キャッシュは既存ユーザーが再訪した際に自動で解放される。テスト: `encodeAlphaVideoWasm.test.ts` を ffmpeg mock 検証から MOV header の byte-level assertion (atom tree 走査、`ftyp` brand `qt  `、`stsd` の `png ` codec + depth 32 + color_table -1、`stsz`/`stco` 整合性、192 frame 想定の実運用サイズ) に書き換え。

### Fixed
- `crates/core/src/variations.rs` の `random_batch_specs()` で count 抽選が `gen_range(usize レンジ)` だったため、rand 0.8 の `UniformInt<usize>` が wasm32(32bit) と native(64bit) で ChaCha8 列の消費ワード数もマッピングも異なり、**同じ seed でも count 抽選を境に RNG 列が割れて orb_size / blur / spec_seed 以降が全 spec で別物になるプラットフォーム依存バグ**を修正 (#242 副産物)。#242 の三者画素比較で「同じ params JSON + 同じ source bytes を食わせているのに CLI(ab_dump) だけ orb 配置が別シーン」になった根本原因（seed=42 / n=12 の spec_idx=8 で native `0xd26a66bedd6c9629` vs wasm32 `0x10cc204f3db50cf9` を実測）。抽選を u32 固定にして native を wasm32 側へ揃えた。**wasm32 では usize = u32 なのでブラウザの既存出力は 1 bit も不変**（本番 GUI の見た目は一切変わらない。`random_batch_specs` の利用者は wasm GUI 経路とテストのみで CLI は不使用）。core 内の usize 抽選はこの 1 箇所のみ（全数確認済み）。ブラウザ実測条件の wasm32 値をピン留めする回帰テスト `random_batch_specs_pins_wasm32_sequence` を追加。

### Changed
- 透過動画出力を ffmpeg.wasm 経由の **PNG codec を MOV container に muxing する経路** に切り替え、ブラウザ/OS/GPU の codec backend に依存せず全環境で安定生成 (#184)。4 段戦略 (CDN 配信 / SW CacheFirst / アイドル時プリフェッチ / saver guard) を以下に分解:
  - **エンコーダ移行**: 旧 WebCodecs `VideoEncoder({codec:'vp09.00.10.08', alpha:'keep'})` 経路 (`web/src/lib/encodeWebmAlpha.ts`) は Edge / Android Chrome / Safari など大多数の環境で `isConfigSupported` が `supported:false` を返し実用にならなかった。新規 `web/src/lib/encodeAlphaVideoWasm.ts` は `@ffmpeg/ffmpeg` を **muxer としてのみ** 使い、PNG codec を QuickTime/MOV container に詰め込む (`-c:v png -pix_fmt rgba -f mov`)。実エンコードは行わず PNG bytes をそのままパッキングするので (a) wasm メモリ枯渇問題なし、(b) アルファプレーン完全保持 (rgba 32bit lossless)、(c) NLE (Premiere / DaVinci / After Effects) 全部対応。経由した失敗: libvpx-vp9 yuva420p (OOM / memory access out of bounds)、libvpx-vp8 alpha (silent に空アルファ)。orberWorker は `animateOneAlpha` / `vp9AlphaSupported` RPC を撤去し、代わりに per-frame PNG を Transferable で main に流す `renderAlphaFrames` を新設。Studio は VP9 alpha probe / `alphaVideoUnsupportedNotice` / `includeAlphaDisabledTitle` を全削除し、`alphaEncodingInProgress` / `alphaEncoderLoadFailed` のみを残す。OFF 経路は引き続き byte-exact identity (ffmpeg.wasm は load されない)。
  - **CDN 配信**: core (~31 MB) は **jsdelivr CDN** (`https://cdn.jsdelivr.net/npm/@ffmpeg/core@<ver>/dist/esm/ffmpeg-core.{js,wasm}`) からバージョン pin で遅延ロード (透過 DL を実際に押した時のみ、1 セッション 1 回)。Cloudflare Pages の単一ファイル上限 25 MiB に `ffmpeg-core.wasm` (~31 MB) が引っかかってデプロイできない問題を回避するため、同一オリジン `/ffmpeg/...` 配信を諦め CDN 経由に切替。`@ffmpeg/core` は `package.json` のバージョン pin 用に `devDependencies` に残すが、`copy:ffmpeg` script は撤去し `dist/` にも `public/ffmpeg/` にも一切バンドルしない。
  - **SW CacheFirst**: Service Worker (`web/public/sw.js`) は `cdn.jsdelivr.net/npm/@ffmpeg/core@` 始まりの URL を CacheFirst で永続化 (`ffmpeg-core-v<version>` キャッシュ、activate で旧バージョン領域を掃除)、初回 fetch 後はオフラインでも透過動画エンコードが継続できる。`cache.put` は `event.waitUntil` で SW lifetime に縛り、cors response のみキャッシュに積む (opaque を弾くことで `importScripts` / WebAssembly streaming compile の CORS チェック失敗を予防)。`cache.match` は `ignoreVary: true` で将来の Vary ヘッダ変更にも耐える。SW 側の `FFMPEG_CORE_VERSION` は build 時の `stamp:sw` で `encodeAlphaVideoWasm.ts` のバージョン定数から注入し二重定義を解消。
  - **アイドル時プリフェッチ + saver guard**: `Studio.tsx` の onMount から `requestIdleCallback` (fallback `setTimeout(2000)`) で `prefetchFfmpegCore()` を発火し、ユーザーがオーブを設計している間に ffmpeg-core (~31 MB) を SW に先取り。プリフェッチは `mode: 'cors'` (cors response のみ受理) で in-flight Promise を保持し、`loadFfmpegAlphaEncoder` がプリフェッチ完走を待ってから `ffmpeg.load` に入ることで二重 fetch race を解消。`navigator.connection.saveData === true` / `effectiveType` が `slow-2g` / `2g` / `3g` のいずれかではプリフェッチを skip し、低速モバイル回線で勝手に 31 MB を取らせない (3G は実 DL 押下時に取ればよい、というトレードオフ)。プリフェッチは singleton (`ffmpegSingleton` / `ffmpegLoadPromise`) には触れず純粋に HTTP fetch のみ走らせるため、失敗しても実 DL 経路は壊れない。

## [0.3.0] - 2026-04-28

### Added
- New CLI flag `--count <N>` (1..=200, default 20) that controls how many orbs are visible on screen at once. The K colors picked by k-means are *expanded* into N orbs by weight-proportional color sampling and per-orb cross-axis scattering, so a single image can fill roughly 70% of the frame at the default count. (#41)
- All-orb breathing modulation in **three independent axes**: radius ±10%, blur ±15%, opacity ±5%. Each orb's three axes are decorrelated by separate seed-derived phase offsets, and each axis loops once per clip duration. (#41)
- `OrbStyle` enum (`Rim` / `Soft`) and `render_one_orb` per-orb rendering helper. Each orb is assigned a style deterministically from the seed (≈50:50), so a single frame mixes the rim-emphasized look with plain soft gradients. (#41)
- Per-orb integer speed multipliers (`1x` / `2x`) assigned deterministically from the seed. Combined with the `MotionSpeed` cycle count (`VerySlow` / `Slow` = 1 / 2), effective traversal counts spread over `{1, 2, 4}` per clip — orbs visibly move at varied paces while pixel-exact loop closure at `t=0 ≡ t=1` is preserved (integer × integer cycles). (#41)
- Off-screen wrap buffer: each orb's progress range is extended from `[0, 1]` to `[-r, 1+r]` (where `r` is its radius normalized by the progress-axis length). Orbs now spawn and despawn fully off-screen, so the wrap moment is invisible — the seam at `pos = 1+r → -r` happens beyond the canvas edges. (#41)
- New cluster helpers `cluster::derive_background_rgba` and `cluster::drop_dominant`. The dominant (highest-weight) cluster becomes the canvas color and is dropped from the orb pool, so the input image's most prevalent color is no longer drawn as an orb on top of itself. (#41)

### Changed
- **BREAKING**: Motion model rebuilt as a one-way conveyor belt. Each clip flows in a single direction (left→right / right→left / top→bottom / bottom→top) with all orbs traveling the same way. Orbs no longer reflect or oscillate; they exit one edge and re-enter from the opposite edge (wrap loop). (#41)
- **BREAKING**: `MotionShape` (`Still`, `Lissajous`, `Vertical`, `Horizontal`, `Diagonal`, `Breathe`, `Twinkle`) is **removed**. The standalone `Breathe` / `Twinkle` modes are gone — breathing is now an automatic effect applied to every clip.
- **BREAKING**: Old `MotionSpeed` variants (`Subtle` / `Slow` / `Lively`) are **removed** in favor of `VerySlow` / `Slow`, defined as integer screen-cross counts per clip (1 / 2). Pixel-exact loop closure at `t=0 ≡ t=1` is preserved.
- **BREAKING**: New `MotionDirection` enum (`LeftToRight`, `RightToLeft`, `TopToBottom`, `BottomToTop`) added.
- **BREAKING**: CLI flags `--motion`, `--motion-shape`, `--motion-speed` are **removed** and replaced with `--direction <lr|rl|tb|bt>` and `--speed <very-slow|slow>`.
- **BREAKING**: Animation boundary mode switched from `clamp` to wrap (`rem_euclid`).
- **BREAKING**: `DEFAULT_VARIATIONS` preset rebuilt around direction × speed × `count` × `orb_size` × `blur` (color is no longer a preset axis). Output filenames change to `snapshot_lr_dense`, `snapshot_rl_huge`, `snapshot_tb_fine`, `snapshot_bt_blurry`, `flow_lr_slow`, `flow_rl_very_slow`, `flow_tb_dense`, `flow_bt_blurry`, `flow_lr_dense_small`, `flow_rl_huge`. (#41)
- **BREAKING**: `VariationSpec` now carries `count: usize` instead of the old color/cluster fields. `VariationSpec.shape` / `VariationSpec.speed` (old types) replaced with `direction` and the new `speed`.
- PNG single-output path now goes through `animate::render_frame(t=0)` instead of `render_static` so `--count` takes effect for stills as well as videos.
- Animated variation duration extended from 4000 ms to 8000 ms so the slow conveyor pacing reads as gentle.

### Removed
- **BREAKING**: `ColorMod` module is **deleted**. Hue shift, lightness bias, saturation modulation, and dominant cluster rotation are gone. The premise — that a single input photo should yield multiple recolored variations — was wrong: if you want different colors, feed in a different image. K-means palette colors are now used unmodified end-to-end. (#41)
- **BREAKING**: `VariationSpec` fields `hue_shift_deg`, `lightness_bias`, `saturation`, `dominant_rotation`, and `cluster_count` are **removed**. The `VariationSpec::color_mod()` method is also gone. The k-means K used by the variations path is fixed internally at 5.
- **BREAKING**: CLI flag `--background` is **removed**, along with the entire `background` module (`Background` enum, `resolve`, `BackgroundParseError`). The background color is now derived automatically from the input image: the dominant (highest-weight) k-means cluster becomes the canvas color, and the remaining clusters become the orb pool. A nightscape gives a black canvas with bright points; a daytime sky gives a sky-blue canvas with floating points; a beige interior gives a beige canvas with small accents. To change colors, change the input image. (#41)
- The `--background transparent` rejection branch for mp4/webm is gone (auto-derived backgrounds are always opaque, so the rejection branch became unreachable). (#41)
